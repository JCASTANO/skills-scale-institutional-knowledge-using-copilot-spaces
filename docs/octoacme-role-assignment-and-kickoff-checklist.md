# OctoAcme — Role Assignment & Kickoff Checklist

## Purpose
Use this checklist at the start of every new project or initiative to confirm that all required roles are assigned, communication channels are set up, key artifacts are in place, and the team has a shared understanding of the delivery cadence.

Complete this checklist during or immediately after the kickoff meeting. The **Project Manager** owns completion; the **Scrum Master** facilitates the kickoff session.

Reference: [Roles and Personas](octoacme-roles-and-personas.md) | [Project Initiation Guide](octoacme-project-initiation.md) | [Project Planning Guide](octoacme-project-planning.md)

## 2. Communication Channels

- [ ] Team chat channel created (e.g., Slack / Teams) and all role owners added
- [ ] Stakeholder distribution list confirmed (email or mailing list)
- [ ] Incident / on-call channel set up (SRE + Customer Support Liaison)
- [ ] Meeting cadence agreed and calendar invites sent:
  - [ ] Daily standup (facilitated by Scrum Master)
  - [ ] Weekly delivery sync (facilitated by Project Manager)
  - [ ] Sprint review / demo (facilitated by Scrum Master / Product Manager)
  - [ ] Retrospective (facilitated by Scrum Master)
- [ ] Escalation path documented (see [Risks & Communication](octoacme-risks-and-communication.md))

---

## 3. Artifacts & Documentation

- [ ] Project board created (Backlog / Ready / In Progress / In Review / QA / Done columns)
- [ ] Project One-pager completed and shared with all role owners
- [ ] Initial risk register created and Risk Register owner assigned (Project Manager)
- [ ] Definition of Done drafted (Scrum Master + QA Lead + Developers)
- [ ] Backlog seeded with initial items, each with an Owner and Reviewer/Approver field
- [ ] QA approach documented (QA Lead)
- [ ] UX design plan and review cadence agreed (UX Designer + Product Manager)
- [ ] Documentation plan created and Technical Writer confirmed (Technical Writer)
- [ ] Repository / project folder structure set up and README updated

---

## 4. Process & Delivery Cadence

- [ ] Sprint length agreed (e.g., 1 week / 2 weeks)
- [ ] Team working norms agreed (e.g., core hours, PR review SLA, async-first policy)
- [ ] Branching and PR conventions documented in the repo
- [ ] CI pipeline configured (tests, lint, security scanning)
- [ ] Deployment process and environments confirmed with SRE
- [ ] On-call rotation set up (SRE) and escalation path documented
- [ ] Release cadence agreed with Product Manager and Project Manager
